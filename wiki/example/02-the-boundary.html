Seam 02 · The one impure ring

The boundary adapter.

The single place the pure core touches the world — and the only object in the system allowed to be impure. It mints identity, reads the clock, stamps the Actor, commits to the bus, and performs effects. Nothing else does any of those things.

▸

The seam at a glance

The eight slots, the same shape as every other seam. Read these, then the deep-dive below.

The seam
The membrane between the pure core and the impure world. Its job: turn a finished agent step into committed, signed, performed reality.
Each side
Inner: the fold and State decide what is true. Outer: the clock, the id source, the append-only log, the effect sinks. The boundary is the one thing that touches both.
What crosses
In: a finished step (tool calls + ToolResult payloads). Out: signed Commands to the bus, and Effect descriptors performed on the world. Forbidden: the fold never receives the clock, an id, or a live sink; a tool never performs an effect.
Direction
Source dependency points inward — the boundary names the fold; the fold names nothing out here. Control flows outward — the boundary drives the world only after the fold has decided.
The inversion
The fold returns Effect descriptors (plain data); the boundary is the adapter that knows how to perform them. The core reaches the world through data it returns, never through a reference to a sink it holds.
Why it exists
To confine all impurity — identity, clock, actor-stamping, commit, effects — to one auditable place, so everything else is pure and the whole session replays deterministically.
What breaks
Mint an id, read the clock, or perform an effect anywhere else and re-fold stops being deterministic — replayTest diverges. Stamp the Actor anywhere else and it can be forged.
The test
The boundary is the one object you test with doubles: a fake clock, a fake id-sequence, a recording sink — then assert the order commit → perform. Everything else is pure and needs none.
A

What it is

When an agent step finishes, the runtime hands the boundary one thing: the step — the tool calls the model made and the ToolResult payloads they returned. The boundary turns that into reality, in a fixed order, and this is the whole of it:

the one impure seam — verbatim from the assembled programpseudocode
boundary.onStepFinish(step):
  now     = clock.read()                                   // (1) injected clock — G9
  results = step.toolResults
  (s, fx) = fold(state, results, now)                      // (2) pure call — decide
  signed  = step.calls.map(c -> sign(c, by = Actor.Agent, id = nextId()))  // (3) mint id + stamp the literal actor
  append(bus, signed, results)                             // (4) COMMIT to the append-only log
  state   = s                                              // (5) adopt the new truth
  for f in fx: perform(f, mode = LIVE, key = f.id)         // (6) the only place effects run

Six lines, and every impure capability in the system is in them: the clock (1), identity and the actor stamp (3), the commit (4), and effects (6). Step (2) — the decision — is a call into the pure fold, which touches none of those things. The boundary is the adapter; the fold is the policy.

B

What crosses it — and the rule that makes a human override free

Two data structures leave the boundary, and both are plain data — never an entity, never a live handle. Outbound to the bus: a signed Command. Outbound to the world: Effect descriptors, performed. The Command is where identity and authorship are minted:

the signed action — what crosses to the buspseudocode
enum Actor { Human, Agent }
sealed Command(by: Actor) {
  SetPriority(by, id: CmdId, ticket: TicketId, level: Priority)
  RequestEscalation(by, id: CmdId, ticket: TicketId)    // a request — reversible
  ConfirmEscalation(by, id: CmdId, ticket: TicketId)    // irreversible — pages only when by is a separate authority (host), never the requesting agent
}

The Actor is stamped here, at the boundary, and nowhere else. This is the rule that makes the architecture's headline move real: because a human tap and an agent tool-call both become a Command at this one seam — identical but for by: Human vs. by: Agent — they flow through the same bus, the same fold, the same replay path. Letting a human do what the agent does is therefore not new code; it is the same command with a different stamp. In A Typical Scenario, swap step 4's Actor.Agent for Actor.Human and every other step is byte-for-byte identical.

The bus the command lands on is append-only and observable — a committed, replayable record. “Commit” at line (4) means exactly this: the signed command and its results are durably appended before any effect fires at line (6). That ordering is not cosmetic; it is what guarantees no effect is ever performed for a decision that was not recorded.

C

Why exactly one impure seam

Clean Architecture puts impurity all around its outer ring: the controller talks to the web, the presenter to the screen, the gateway to the database. Each is a separate place where the world leaks in. This architecture makes a sharper promise: collapse every one of those into a single object, and make it the only impure object in the system.

The payoff is that purity becomes a property you can state in one sentence and check at one place. Identity and time — the two things most likely to make a program non-reproducible — are injected into the boundary (the clock is read at line 1, ids are minted at line 3) and passed into the fold as ordinary values. The fold cannot reach for a clock or a random id because it was never handed the means to. That single discipline is what lets the recorded command stream re-fold to a bit-identical state every time.

The order is the contract

Read lines (4)→(5)→(6) again: commit, then adopt, then perform. If a crash happens between commit and perform, recovery re-reads the committed stream, re-folds to the same state, and re-issues the same effects — which the sinks dedupe by key = f.id. The boundary's job is not just to be the one impure place; it is to be impure in a fixed, recoverable order.

D

What breaks without it

The seam earns its keep only against the version that omits it. Here is the most tempting violation — a tool that just does the work itself:

Collapsed — the tool performs the effect

DOMAIN_TOOL setPriority {
  run(input, ctx) -> { db.write(input); pager.page(input) }  // impure!
}

It looks shorter and it works on the happy path. But the tool now performs an effect, so it is no longer a pure function of its input. The test that dies: replayTest can no longer run off-target — re-folding the recorded stream re-hits db and pager, double-writing and re-paging, and the “fold twice, assert identical” check is meaningless because folding now has side effects. You have not saved code; you have deleted replay.

Collapsed — the surface stamps the actor

If the UI mints Command.SetPriority(by = Agent, ...) for an action a person took — or vice-versa — the Actor is now forgeable. The audit log says “the agent did this” when the host did, replay reconstructs the wrong authorship, and the “human override is free” guarantee collapses into “human override is a second, untrusted code path.” Stamping at one seam is what keeps authorship honest.

Both failures share a shape: a capability that belongs to the boundary leaked somewhere it can be neither audited nor replayed. The seam exists precisely to make that leak impossible to write by accident.

E

The test that the seam makes possible

Because the boundary is the only impure object, it is also the only one you ever test with doubles. Inject a fake clock, a fake id-sequence, and a recording sink, then assert the order of what came out:

testing the one impure objectpseudocode
test "commit precedes perform, actor is stamped once":
  clock = FakeClock(t0); ids = FakeIds([c1]); sink = RecordingSink()
  boundary.onStepFinish(step = oneCall(setPriority, ticket = 4118, level = High,
                                       actor = Agent))
  assert sink.log == [
    Append(Command.SetPriority(by = Agent, id = c1, ticket = 4118, level = High), result = PrioritySet(4118, High)),  // commit = command + its result
    Perform(Effect.Log("priority", 4118, High, at = t0))      // commit BEFORE perform
  ]

Everything else in the program — the fold, the tools, the projection — is pure and is tested by direct call, with no doubles at all. That asymmetry is the whole point: one object is hard to test, so it is made small and tested carefully; everything else is made pure, so it is trivial to test. That is the Humble Object pattern, and the boundary adapter is its sharpest instance in this architecture.
