00

What this is, and how it teaches

A single small application, documented seam by seam. The application is a teaching device; the method is the point — borrowed, deliberately, from the most widely-read book on software boundaries.

The running example. A support-ticket triage console. An agent reads an incoming ticket stream, drafts replies, sets priority, and can request escalation; a rare human operator — the host — can take the same actions through the same commands; and the customer receives a resolution without ever touching the interface. It is the same illustrative scenario the reference uses. Nothing in the architecture depends on it — substitute your own domain freely.

The method, borrowed from Clean Architecture. That book does not catalogue boundaries abstractly; it traces a single request across each one in turn, names a concrete participant at every hop, and hands a simple data structure across each seam. We do the same. First the picture and the one law (below); then, for readers who already know Clean Architecture, the exact mapping of its vocabulary onto ours (§ mapping); then a single command traced end to end (§ a typical scenario); then the whole program assembled (§ the whole example); and finally the seam pages, each running the same eight-slot template (§ how to read a seam).

The one move worth holding in mind

A human action and an agent action are the same signed Command on one append-only, replayable stream, differing only by an Actor label. Every seam below exists to keep that true. When a seam's purpose is unclear, ask: what would stop being replayable, or pure, or overridable, if this seam were removed? That question — “what breaks without it” — is the eighth slot of every seam page.

01

The rings, and the one law

Clean Architecture draws concentric rings and asserts a single rule over them: source-code dependencies may only point inward, toward higher-level, more stable policy. This architecture keeps that rule and sharpens it.

The law, in one sentence. Purity points inward, and only the boundary adapter is impure. Concretely: the State and the fold import nothing foreign; tools, the projection, and the port interfaces are pure and depend only on the core; and every impure act — minting identity, reading the clock, stamping the actor, committing to the bus, performing an effect — is confined to a single seam. Clean Architecture scatters impurity across its entire outer ring (controllers, presenters, gateways, the database, the web). This architecture collapses all of it into one named place. That is the sharper, more teachable invariant, and it is the spine of everything that follows.

Dependency direction vs. flow of control

The two run opposite ways, and the difference is the whole trick. At runtime, control flows outward — the fold hands effect descriptors out to the boundary, which performs them on the world. But every source dependency points inward — the boundary names the fold; the fold names nothing outer. The device that lets control go one way while the compile-time arrow goes the other is a polymorphic interface declared on the inner side: our ports and our effect descriptors. Hold this; it recurs at every seam.

Three identifications are worth stating plainly. One is exact, point for point: our project is the Presenter and our surface is the View — the Humble Object split, unchanged. The other two map onto Clean Architecture and then sharpen it: our domain port and effect descriptors play the role of the Input/Output Ports — the polymorphic interfaces that make the source dependency oppose the flow of control, though the reference models them as one DomainPort rather than a split pair; and our boundary adapter collapses the whole outer adapter ring into one seam — the only impure one, a deliberate divergence from Clean Architecture's scattered outer ring rather than an analogy to it.

03

A typical scenario

One event, traced across every seam in sequence, a named data structure handed across each crossing. This is the spine the seam pages then deepen, one crossing at a time.

The event: a new ticket, #4118, arrives in the queue. The agent reads it and sets its priority to High. Follow the thread:

  1. Sensing (EventSource port). The TicketQueue adapter receives a raw event and normalizes it to a typed domain event, TicketArrived(4118, body). Raw event in; no domain logic here.
  2. The step + agent context. The boundary stages the event into a read-only ctx for the next step; the serial consumer runs one turn. The agent reasons and chooses a tool.
  3. The tool crossing. It calls setPriority(ticket=4118, level=High). The tool's run(input, ctx) reads ctx, returns the payload PrioritySet(4118, High), and mutates nothing. pure
  4. Identity & actor (boundary adapter). The boundary mints an id, reads the injected clock, and stamps Actor.Agent, building Command.SetPriority(by=Agent, id, 4118, High). The actor is stamped here and nowhere else.
  5. The command bus. append(bus, signed, result) commits the command and its result to the append-only, observable log. Nothing happens off the record.
  6. The fold. fold(state, [PrioritySet(4118,High)], now) returns (state', [Effect.Log(...)]) — a new immutable state and an effect descriptor. It performs nothing. pure
  7. The effect, performed (boundary adapter). perform(Effect.Log(...), LIVE). The one impure act in the whole trace.
  8. The projection & surface. project(state') recomputes the view: ticket 4118's row now reads badge:"HIGH", canEscalate:true. The surface renders it. It decides nothing — it applies flags already decided. pure

Notice the asymmetry made concrete: at steps 4–7 control flows outward (tool → boundary → world), while every source dependency along the way points inward at the fold and the State. That is the law of §01, walked.

04

The whole example, assembled

Every fragment the seam pages dissect, in one place, in neutral pseudocode. This is the canonical source: the names here are the names everywhere. Copy it, then read the seams to learn why each line sits where it does.

support-ticket triage console — the whole programpseudocode
// ───────── Entities · the inner ring · pure, no I/O (seam 01) ─────────
enum Priority { Low, Normal, High, Urgent }
sealed TicketStatus { Open | Escalating(by: Actor) | Escalated | Resolved }
type  Ticket = { id: TicketId, body: Text, priority: Priority, status: TicketStatus }
sealed State = Console { tickets: Map<TicketId, Ticket>, run: RunStatus }
sealed RunStatus { Idle | Working(step) | Degraded(cause) | Error(fault) }

// ───────── The signed action · the DTO that crosses the bus (seam 02) ─────────
enum Actor { Human, Agent }
sealed Command(by: Actor) {
  SetPriority(by, id: CmdId, ticket: TicketId, level: Priority)
  RequestEscalation(by, id: CmdId, ticket: TicketId)    // a request — reversible
  ConfirmEscalation(by, id: CmdId, ticket: TicketId)    // irreversible — pages only when by is a separate authority (host), never the requesting agent
}

// ───────── Tool results · the DTOs a tool returns (seam 03) ─────────
sealed ToolResult {
  PrioritySet(ticket: TicketId, level: Priority)
  EscalationRequested(ticket: TicketId, by: Actor)
  EscalationConfirmed(ticket: TicketId, by: Actor)     // carries WHO confirmed — the gate reads it
  Focused(ticket: TicketId, known: Bool)
}

// A DOMAIN TOOL — pure. Reads read-only ctx, returns a payload, mutates nothing.
DOMAIN_TOOL setPriority {
  input = { ticket: TicketId, level: Priority }
  run(input, ctx) -> PrioritySet { ticket: input.ticket, level: input.level }
}
// A UI TOOL — also pure, also returns a payload. Drives the surface, decides nothing.
UI_TOOL focusTicket {
  input = { ticket: TicketId }
  run(input, ctx) -> Focused { ticket: input.ticket, known: ctx.tickets.has(input.ticket) }
}

// ───────── The fold · the Use-Case Interactor · pure (seam 01) ─────────
// Returns (newState, effects). It RETURNS effect descriptors; it never performs them.
fold(state, results, now) -> (State, [Effect]):
  for r in results:
    match r:
      PrioritySet(ticket, level) ->
        ( state.withPriority(ticket, level),
          [ Effect.Log("priority", ticket, level, at = now) ] )
      EscalationRequested(ticket, by) ->
        ( state.withStatus(ticket, Escalating(by)),
          [] )                                  // a request commits no irreversible effect
      EscalationConfirmed(ticket, by) ->                        // the gate: only a host confirm pages
        by == Human
          ? ( state.withStatus(ticket, Escalated), [ Effect.PageOncall(ticket, at = now) ] )
          : ( state, [ Effect.Diag("denied: agent self-confirm") ] )   // agent-stamped confirm → no page
      else ->
        ( state.withUnhandled(r), [ Effect.Diag(r) ] )

// Effect descriptors — plain data. Performed only at the boundary (seam 01 → 02).
sealed Effect { Log(...) | PageOncall(...) | Diag(...) }

// ───────── The boundary adapter · the ONE impure seam (seam 02) ─────────
boundary.onStepFinish(step):
  now     = clock.read()                                   // injected clock (G9)
  results = step.toolResults
  (s, fx) = fold(state, results, now)                      // pure call
  signed  = step.calls.map(c -> sign(c, by = Actor.Agent, id = nextId()))  // Actor minted HERE (a literal) — a human tap dispatches with Actor.Human
  append(bus, signed, results)                             // COMMIT to the append-only log
  state   = s
  for f in fx: perform(f, mode = LIVE, key = f.id)         // the only place effects run

// ───────── The view projection · the Presenter · pure (seam 04) ─────────
project(state) -> ConsoleView:
  ConsoleView {
    rows: state.tickets.values.map(t -> TicketRow {
      ticket:      t.id,
      badge:       badgeFor(t.priority),        // "HIGH" + colour, pre-computed
      canEscalate: t.status == Open,            // the flag is decided HERE, not in the view
      escalating:  t.status is Escalating }),
    banner: bannerFor(state.run)
  }

// ───────── The surface · the View · humble (seam 04) ─────────
render(view: ConsoleView):
  for row in view.rows: drawRow(row)            // no `if` over data, no computed flag
onAction(intent) -> dispatch(intent)            // one sink; the boundary signs it

// ───────── The ports · interfaces the core owns (seam 05) ─────────
PORT Runtime       { dispatch(Command); observe() -> Stream<State> }   // the domain port
PORT ModelProvider { step(prompt, tools) -> StepResult }              // the model backend
PORT EventSource   { events() -> Stream<RawEvent> }                    // sensing (tickets in)

// ───────── The composition root · the only wiring site (seam 06) ─────────
wireApp(env):
  bus     = Bus()
  store   = LogStore(env.path)
  model   = env.live ? HttpModel(env.url) : ScriptedModel(env.fixtures)   // ← the swap
  sensing = TicketQueue(env.queue)
  runtime = Runtime(model, bus, store, boundary)
  return App(runtime, sensing)

// ───────── The replay harness · tests are the outer ring (seam 07) ─────────
replayTest(recorded):                           // recorded = signed commands + fixtures
  a = foldAll(initialState, recorded)
  b = foldAll(initialState, recorded)
  assert a.state == b.state                      // determinism (G9)
  assert collectPerform(recorded, mode = REPLAY) == recorded.goldenEffects

05

How to read a seam

Every seam page — all seven — runs the same eight-slot template. Learn the slots once; then each page is a five-minute read and the uniformity does the teaching.

The seam
Its name, and a one-line statement of the conversation it carries.
Each side
The responsibility on the inner side and on the outer side — policy vs. mechanism, in one sentence each.
What crosses
The exact data structure that may cross — and, loudly, the forbidden payload that may not (an entity, a live handle, the clock).
Direction
Which way the source dependency points (inward) and which way control flows (often outward).
The inversion
The polymorphic interface, declared on the inner side, that lets control oppose the dependency.
Why it exists
The volatility it quarantines, or the property (replay, purity, override) it protects.
What breaks
The concrete failure if you violate it — shown as a collapsed anti-example, with the test that dies.
The swap / the test
What becomes testable once the seam exists — usually: the whole session replays off-target.
