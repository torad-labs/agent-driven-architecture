Seam 06 · Slices and wiring

Blocks and the root.

Two seams in one page. A block is a self-contained feature — a mini-hexagon, Clean Architecture's partial boundary — whose only public symbol is its tool registration. The composition root wireApp(env) is Main: the single place, in the whole program, where concrete adapters are chosen. A block contributes typed pieces up into one shared spine; the root passes concrete adapters down.

▸

The seam at a glance

The eight slots — but read as two seams that frame each other: the block (a bundle of seams behind one registration) and the root (the one dirty wiring place). Read these, then the deep-dive below.

The seam
A block: a vertical slice that bundles many seams behind a single public symbol — its tool registration. And the root: the one site, wireApp(env), where concrete adapters are chosen. The seam is the line a block draws around itself; the root is the line Main draws across all layers.
Each side
Inner: the spine — one bus, one fold, one boundary (the trunk). Outer / leaf: the block — a namespaced State slice, its fold arms, a projection, a port and its private adapter (the leaves). A block plugs into the trunk; it is never the trunk.
What crosses
Up (block → spine): a tool, a State-slice shape, fold arms, and Command cases — typed pieces, contributed into the shared spine. Down (root → block): concrete adapters, chosen once. Forbidden: a block importing a sibling's internals, or instantiating a second bus, fold, or boundary — a second baseplate.
Direction
A block depends only on shared domain types, the bus, and the folded State — never on a sibling, never outward at all. Only the root spans layers: it alone names HttpModel, LogStore, TicketQueue. Everything else points inward.
The inversion
Two blocks couple only through shared State and the bus — never by import. The escalation block reads “ticket #4118 is High” from the folded State the priority block wrote; it never reaches into the priority block to ask. State is the wire; import is forbidden.
Why it exists
A block is a partial boundary you can promote to a hard one later — a place held open for change at low cost. The root is the one deliberately dirty seam (Main), so that dirt is named and contained instead of smeared across every feature.
What breaks
A block that reaches into a sibling's slice by import couples them into one unit that can no longer move. A block that stands up its own bus creates a second mutation path — a write replay never sees. The check that dies: replayTest, because the recorded stream no longer reconstructs the truth.
The test
A block is tested in isolation against its contract: feed its tool, fold its arms, assert its slice and effects — no siblings, no root, no live adapters. The root is exercised by the replay harness, which wires the whole graph and folds the recorded stream end to end.
A

What a block is — a mini-hexagon behind one symbol

The seam pages so far have dissected the spine: one State and fold, one boundary, one bus. But a real application is not one feature — it is many, and they must be added, tested, and removed without each touching the others. The unit that makes that possible is the block: a self-contained feature, a mini-hexagon, whose entire public surface is a single symbol — its tool registration. Everything else it owns is private.

A block contributes four typed pieces into the shared spine, and nothing more: a tool (or a small set), a namespaced State slice it alone writes, fold arms that translate its ToolResults into that slice, and the Command cases its actions sign. It owns, privately, a projection over its slice and — when it reaches the world — a port with a private adapter behind it. Siblings see none of that. They see one registered tool name.

Treat escalation as a block. Its public symbols are the tools requestEscalation and confirmEscalation; its slice is the per-ticket TicketStatus; its fold arms turn EscalationRequested / EscalationConfirmed into status transitions; and behind a private PageOncall port sits the adapter that actually pages. It plugs into the same spine as setPriority — same bus, same fold, same boundary — and it knows nothing about the priority block at all.

an escalation block — only its registration is public; everything else is private to the blockpseudocode
// ── BLOCK: escalation ─────────────────────────────────────────────
//   public surface = ONE symbol: register(). Everything below is private.

block Escalation {

  // (1) PUBLIC — the only export. Contributes tools UP into the spine.
  register(spine):
    spine.addTools([ requestEscalation, confirmEscalation ])   // tools
    spine.addFoldArms(escalationArms)                          // fold arms
    spine.addCommandCases([ RequestEscalation, ConfirmEscalation ])  // Command cases
    spine.declareSlice(status: Map<TicketId, TicketStatus>)    // a State-slice SHAPE

  // (2) PRIVATE — the block's own tools. Pure; return payloads (seam 03).
  private requestEscalation { input = { ticket }
    run(input, ctx) -> EscalationRequested { ticket: input.ticket, by: ctx.actor } }
  private confirmEscalation { input = { ticket }                // GATED: host-only (Actor.Human), never agent-self-served
    run(input, ctx) -> EscalationConfirmed { ticket: input.ticket, by: ctx.actor } }   // records the confirmer

  // (3) PRIVATE — fold arms over THIS block's slice only (seam 01).
  private escalationArms(state, r, now):
    match r:
      EscalationRequested(ticket, by) -> ( state.withStatus(ticket, Escalating(by)), [] )
      EscalationConfirmed(ticket, by) -> by == Human                                            // the gate — agent self-confirm denied
                                           ? ( state.withStatus(ticket, Escalated), [ Effect.PageOncall(ticket, at = now) ] )
                                           : ( state, [ Effect.Diag("denied: agent self-confirm") ] )

  // (4) PRIVATE — the block's port + its own adapter (seam 05). Siblings never see it.
  private PORT PageOncall { page(ticket) }
  private adapter LivePager implements PageOncall { ... }   // chosen by the ROOT, not here
}

Read what is and is not exported. Out of the block: a tool name, a slice shape, fold arms, Command cases. Kept inside: the tool bodies, the fold arms' internals, the projection, the port, the adapter. A sibling cannot name escalationArms or PageOncall — they do not exist outside the block. That asymmetry is the partial boundary: a feature you can reason about, test, and later harden as a unit.

B

How blocks plug into one spine

A block never stands up its own infrastructure. It does not new Bus(), it does not write its own fold loop, it does not own a boundary. Those are the spine — one each, shared — and the block plugs into them through the typed pieces it registers. The priority block and the escalation block both contribute up into the same trunk; the root reaches down and hands the trunk its concrete adapters.

The slice is the heart of it. Each block owns a namespaced region of the one State: the priority block writes tickets[id].priority, the escalation block writes tickets[id].status. They share the same State value and the same fold loop, but their fold arms touch disjoint fields. So a block reads what a sibling produced the only honest way — off the folded State — and writes only its own region. The bus carries the signed commands; the State carries the result; neither block ever imports the other.

A block is a partial boundary — by design

Clean Architecture distinguishes a full boundary (a polymorphic interface with two implementations, fully decoupled) from a partial one (a slice you have isolated but not yet fully inverted). A block is the partial kind on purpose: it is held open at the seam — one public symbol, a private interior — so that if the feature earns it, you promote it to a hard boundary later. You do not pay for the full boundary up front. You pay for the line that lets you draw the full one cheaply when the day comes.

C

The composition root — the one place adapters are chosen

If a block never names a concrete adapter, someone must. That someone is the composition root, wireApp(env) — this architecture's Main. It is the single function in the program that spans layers: it constructs the spine, picks each port's concrete implementation from the environment, and hands them down. Every new of an impure thing lives here and nowhere else.

the composition root — verbatim from the assembled program (seam 06)pseudocode
wireApp(env):
  bus     = Bus()
  store   = LogStore(env.path)
  model   = env.live ? HttpModel(env.url) : ScriptedModel(env.fixtures)   // ← the swap
  sensing = TicketQueue(env.queue)
  runtime = Runtime(model, bus, store, boundary)
  return App(runtime, sensing)

Six lines, and they are the only six allowed to be dirty. Notice what they do. They build the spine once — one Bus, one LogStore, one Runtime wrapping the one boundary. They make the swap — HttpModel live, ScriptedModel under test — at the one site a swap is legal. And they choose the sensing adapter, TicketQueue, for the EventSource port. A block's private PageOncall would be satisfied here too: the root is where LivePager meets the escalation block's port. The blocks declared the ports; the root, alone, decides what fills them.

This is the whole reason the rest of the program can be pure. Push every construction decision to one function and the dependency arrows everywhere else point inward — toward interfaces and shared types — because nothing else is allowed to know a concrete name. The root is the sink for all the impurity that the seams quarantined: it spans layers so that nothing else has to.

D

What breaks without it

Two violations dissolve the block boundary, and both are tempting because they look like less work. The first reaches sideways; the second builds a second trunk.

Collapsed — the escalation block imports the priority block's internals

block Escalation {
  import Priority.priorityArms  // reaches into a sibling's interior
  run(input, ctx) -> if Priority.levelOf(ctx, input.ticket) == Urgent: ...
}

It compiles, and it even reads naturally. But the two blocks are now one unit: you cannot test escalation without dragging priority in, cannot move or delete either, and the “single public symbol” line is gone. The block was a partial boundary you could promote; now it is a knot you must first untie. The honest path was already available — read the level off the shared, folded State, which the priority block wrote there for exactly this. The test that dies: the block-in-isolation test — escalation can no longer be exercised against its contract alone, because its contract now secretly includes the priority block.

Collapsed — the block stands up its own bus (a second baseplate)

block Escalation {
  private bus2 = Bus()  // a second mutation path
  run(input, ctx) -> bus2.commit(EscalationConfirmed(input.ticket))
}

Now there are two append-only logs, and the truth is split across both. The replay harness reads the one recorded stream — the spine's bus — folds it, and gets a State that is missing every escalation the block committed to bus2. The test that dies: replayTest — foldAll(initialState, recorded) no longer reconstructs the live state, because a second baseplate created a write the recording never saw. One bus is what makes the session a single replayable fact; a second bus deletes that guarantee.

Both failures share the spine's shape inverted: a capability that belongs to the one trunk — a sibling's interior, or the single bus — got duplicated or breached inside a leaf. The block boundary exists precisely so neither is writable by accident: a block can only reach a sibling through State, and can only commit through the spine it was handed.

E

The two tests the seam makes possible

The block and the root are tested at opposite scales, and that split is the payoff. A block is tested in isolation against its contract — feed its tool, fold its arms, assert its slice and the effects it emitted — with no siblings, no root, and no live adapters in sight:

The confirm is gated — the agent may request, only a separate authority confirms

One fidelity point the block must honour: requestEscalation is agent-callable and reversible, but confirmEscalation fires the irreversible PageOncall, so it is gated. Because the Actor is stamped at the boundary (seam 02), an agent acting through its own step is stamped Actor.Agent and cannot mint a host confirm. The fold's confirm arm then pages only when by == Human — a separate confirming authority (the host here; in an unattended deployment a policy tier or reviewer on the same bus) — and folds an agent-stamped confirm to a no-op Diag, never a page. A single bad inference can ask to page on-call, but cannot page on-call. The test below exercises both directions.

a block tested against its contract — no siblings, no root, no live adapterspseudocode
test "escalation block: a HOST confirm transitions to Escalated and pages once":
  r       = confirmEscalation.run({ ticket: 4118 }, ctx = { actor: Human })   // out-of-band host confirm
  (s, fx) = escalationArms(stateWith(4118, Escalating(Agent)), r, now = t0)    // the agent had requested
  assert s.statusOf(4118) == Escalated                         // its slice, alone
  assert fx == [ Effect.PageOncall(4118, at = t0) ]            // its effect descriptor
  // and the gate holds the other way — an agent-stamped confirm pages nobody:
  rA       = confirmEscalation.run({ ticket: 4118 }, ctx = { actor: Agent })
  (_, fxA) = escalationArms(stateWith(4118, Escalating(Agent)), rA, now = t0)
  assert fxA == [ Effect.Diag("denied: agent self-confirm") ]   // no PageOncall — self-confirm denied
  // no priority block, no bus, no wireApp — the block stands on its own contract

The root, by contrast, is the seam the replay harness exercises: it is the only thing that wires the whole graph, so it is verified by folding the whole recorded stream and asserting the session reconstructs. The two scales never overlap — a block proves its contract; the root proves the composition.

That is the same Clean-Architecture lesson the spine taught, applied one level up. There, one object was made impure so the rest could be pure. Here, one function — Main — is made cross-layer so every block can stay a single-symbol, inward-pointing slice. Concentrate the spanning in one named place, and everything around it gets small, testable, and free to move. A block is the vertical slice; wireApp(env) is Main; and keeping them honest is what lets the program grow feature by feature without the features fusing.
