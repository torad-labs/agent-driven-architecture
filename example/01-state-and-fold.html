Seam 01 · The inner ring

State and the fold.

The centre of the diagram, and the seam everything else is built to protect. The State is the enterprise truth — a discriminated union. The fold is the pure reducer fold(state, results, now) that turns recorded tool results into a new immutable State plus Effect descriptors. It reads no clock, mints no id, performs no I/O — which is exactly why the whole session replays.

▸

The seam at a glance

The eight slots, the same shape as every other seam. Read these, then the deep-dive below. This page covers three tightly-bound seams — State, the fold, and Effect — with the fold as the primary one: State is its in and out, effects are what it emits.

The seam
The pure reducer fold(state, results, now) -> (newState, effects) — the use-case layer. It folds recorded ToolResults into a new immutable State and a list of Effect descriptors. Nothing else decides what is true.
Each side
Inner: State, Ticket, Priority — the entities, the enterprise truth, a discriminated union. Through it: the fold reads the current State and the results, and produces the next State. The fold imports nothing foreign; the entities import nothing at all.
What crosses
In: the current state, a list of ToolResult, and now — a clock reading passed as an ordinary value. Out: a new immutable State and a list of Effect descriptors (plain data). Forbidden: the fold never reads a clock, never mints an id, never performs an effect, never does I/O. now is handed in; effects go out as data.
Direction
Source dependency points inward — the fold names State and the entities, and nothing outer; the entities name nothing. Control flows outward — the fold returns Effect descriptors that the boundary will later perform.
The inversion
The fold emits Effect descriptors — plain data — instead of performing the effect. That is the Output Port: the core reaches the world through data it returns, never through a sink it holds. The boundary is the adapter that performs them.
Why it exists
Purity. Because the fold is a pure function of (state, results, now), the recorded command stream re-folds to a bit-identical state every time — the whole session replays — and the fold is tested by a direct call, with no doubles.
What breaks
Let the fold read the clock or perform an effect and re-folding stops being deterministic. replayTest's “fold twice, assert identical” becomes meaningless — folding now has side effects, and the second fold double-writes. You have not saved a line; you have deleted replay.
The test
Assert fold(s, results, t0) == fold(s, results, t0) — pure, no doubles, no fakes. And assert that effects are returned, not performed: the fold's output is a tuple of data, and the world is untouched until the boundary runs it.
A

What it is

At the centre of the rings sits the State — the Clean-Architecture Entities: the enterprise truth, the data and rules that would still be true if there were no agent, no screen, no network. It is modelled as a discriminated union, not as a bag of flags, so that an illegal state cannot be named in the first place. A ticket is Open or Escalating or Escalated or Resolved — never two at once, never none.

the entities — verbatim from the assembled programpseudocode
enum Priority { Low, Normal, High, Urgent }
sealed TicketStatus { Open | Escalating(by: Actor) | Escalated | Resolved }
type  Ticket = { id: TicketId, body: Text, priority: Priority, status: TicketStatus }
sealed State = Console { tickets: Map<TicketId, Ticket>, run: RunStatus }
sealed RunStatus { Idle | Working(step) | Degraded(cause) | Error(fault) }

Around that truth runs the fold — the Clean-Architecture Use-Case Interactor, expressed as a pure reducer. When an agent step finishes, the recorded ToolResult payloads are handed to the fold together with the current state and a single clock reading, now. The fold matches each result against the union and returns the next truth plus the effects that truth implies — and this is the whole of it:

the pure reducer — the use-case layer, verbatimpseudocode
fold(state, results, now) -> (State, [Effect]):
  for r in results:
    match r:
      PrioritySet(ticket, level)      -> ( state.withPriority(ticket, level), [ Effect.Log("priority", ticket, level, at = now) ] )
      EscalationRequested(ticket, by) -> ( state.withStatus(ticket, Escalating(by)), [] )                         // request: no irreversible effect
      EscalationConfirmed(ticket, by)  -> by == Human                                           // the gate: only a host confirm pages
                                            ? ( state.withStatus(ticket, Escalated), [ Effect.PageOncall(ticket, at = now) ] )
                                            : ( state, [ Effect.Diag("denied: agent self-confirm") ] )
      else                            -> ( state.withUnhandled(r), [ Effect.Diag(r) ] )

Read the four arms as policy, not plumbing. PrioritySet writes the new priority and asks for a log line stamped at = now. EscalationRequested moves the ticket to Escalating and returns no effect — a request is reversible, so nothing irreversible may fire. Only EscalationConfirmed returns Effect.PageOncall — and only when it carries a separate authority (by == Human); an agent-stamped confirm folds to a no-op Diag. The page is irreversible, so it is named only after an explicit, separately-authorized confirm. The fold encodes the rule; the boundary later performs the consequence. Both clock-stamps — at = now — use the value that was handed in; the fold never reaches for a clock of its own.

B

What crosses it — effects as descriptors, not actions

The fold's output is a tuple: (newState, effects). The first element is the next enterprise truth — a fresh immutable State, never a mutation of the old one. The second is a list of Effect values, and these are the crossing that makes the whole architecture work:

effect descriptors — plain data the fold returnspseudocode
sealed Effect { Log(...) | PageOncall(...) | Diag(...) }

An Effect is a description of a thing to do, not the doing of it. Effect.PageOncall(ticket, at = now) is a small value that says “page on-call about this ticket”; it does not open a connection, it does not ring a phone. The fold returns it and is finished. Later, at exactly one seam — the boundary adapter — that value is matched and performed against the world. This is the Output Port of Clean Architecture, realised as data: the inner ring reaches outward by returning a description, never by holding a reference to the sink that will act.

Why data and not an action? Because data can be recorded, compared, and replayed; an action cannot. An Effect.Log in a golden file is a fact you can assert against. A call to logger.write(...) buried inside the fold is a side effect you can only observe by spying. The architecture chooses the version you can hold still and look at — and that choice is the line between a session that replays and one that merely ran once.

Returns out, never reaches out

Trace the arrows. The current state and the results come in as arguments; the new State and the Effect list go out as the return value. At no point does the fold hold a handle to a database, a clock, or a sink. Control will flow outward — those effects do reach the world — but the source dependency points inward: the fold names State and Effect, both of which live in the inner ring, and names nothing in the outer ring at all.

C

Why pure — entities stable, now injected, illegal states unrepresentable

Purity here is not an aesthetic; it is three concrete disciplines, each buying a concrete property.

The entities are stable. State, Ticket, and Priority are the highest-level, least-volatile policy in the system — the part that changes only when the business changes, never when a framework or a screen does. Clean Architecture puts exactly this at the centre and forbids it from depending on anything outer. The fold sits one ring out and depends only on it. Nothing here knows that tickets arrive over a queue or that effects end up on a pager; that knowledge lives in rings the fold cannot see.

now is injected, not read. Time and identity are the two things most likely to make a program irreproducible, so neither is available to the fold. The clock is read once, far outside, and its value travels in as the ordinary argument now; both clock-stamps in the body — at = now — use that argument. The fold cannot reach for the wall clock because it was never handed the means to. Pass the same t0 twice and the two folds are indistinguishable, byte for byte.

The union makes illegal states unrepresentable. Because TicketStatus is a sealed discriminated union, the match in the fold is exhaustive: every arm is a real, reachable status, and a status that does not exist cannot be matched, constructed, or fallen into. There is no “escalated and open” to guard against, because the type cannot spell it. The fold's correctness is therefore a property the compiler checks, not a set of runtime ifs the reader must trust. Model the truth as a union and whole classes of bug stop being writable.

The payoff, in one sentence

Stable entities + injected now + an exhaustive union means the fold is a mathematical function of its arguments. Same inputs, same outputs, forever — which is precisely the precondition that lets the recorded stream of seam 07 re-fold to a bit-identical State. Purity is not a virtue the fold has; it is the mechanism replay is built on.

D

What breaks without it

The seam earns its keep only against the version that omits it. Here is the most tempting violation — a fold that, instead of returning a descriptor, just does the work and reads the clock itself:

Collapsed — the fold performs the effect and reads the clock

fold(state, results) {  // note: no `now` argument
  for r in results: match r {
    PrioritySet(ticket, level) -> {
      db.write(ticket, level); logger.log("priority", ticket, level, at = now())  // impure!
      state = state.withPriority(ticket, level)
    } } }
}

It looks shorter and it works on the happy path. But the fold now reads now() and calls db.write and logger.log, so it is no longer a pure function of its arguments. The test that dies: replayTest. Its core assertion — fold the recorded stream twice, assert the two results are identical — becomes meaningless, because the second fold reads a different now() and re-hits db and logger, double-writing the row and re-logging the line. assert a.state == b.state may even pass by luck while the world has been mutated twice behind its back. You have not saved a line of code; you have deleted replay, and with it every off-target test the architecture promised.

The failure has a single shape: a capability that belongs to the boundary — reading the clock, touching a sink — leaked into the inner ring, where it can be neither passed in deterministically nor replayed. The seam exists precisely to make that leak impossible to write by accident: the fold's signature simply has nowhere to reach a clock from, and its return type carries effects out as data.

E

The test the seam makes possible

Because the fold is pure, it is the easiest thing in the program to test: a direct call, no doubles, no fakes, no setup. Determinism is the assertion, and the assertion is one line:

testing the pure reducer — direct call, no doublespseudocode
test "the fold is deterministic and returns effects, never performs them":
  results = [ PrioritySet(ticket = 4118, level = High) ]
  (a, fxA) = fold(initialState, results, now = t0)
  (b, fxB) = fold(initialState, results, now = t0)
  assert a == b                                            // same inputs -> same State, every time
  assert fxA == fxB == [ Effect.Log("priority", 4118, High, at = t0) ]   // effects are RETURNED, as data
  assert world.untouched()                                // nothing performed: no db write, no page, no log

Two things are proven and neither needs a test double. First, determinism: fold(s, results, t0) == fold(s, results, t0), because the only source of time is the argument t0 and there is no hidden clock to diverge. Second, effects are returned, not performed: fxA is a list of plain Effect values you assert against directly, and the world is untouched — no page has fired, no row has been written — because the fold has no means to fire one. The boundary alone will perform these later.

That is the asymmetry the architecture is built on, seen from the inside. The one impure object — the boundary — is tested with fakes and is kept small. Everything in the inner and middle rings — the fold here, the tools, the projection — is pure, and is tested by direct call. State is the truth; the fold is the only thing allowed to change it; and because the fold is pure, the whole session of changes replays. Every later seam exists to keep that sentence true.
